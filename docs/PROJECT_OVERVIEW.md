# 老布偶猫量化工作台 - 项目介绍

## 一句话定位

一款把市场观察、量化策略、真实持仓、AI 多角色研究与复盘串成闭环的个人投资工作台。

## 面试展示口径

- GitHub Pages 提供脱敏静态交互演示，用于验证信息架构和关键操作路径。
- PDF 使用本地真实系统截图，用于说明实际实现状态。
- 动态后端不公开暴露，只在本地或经批准、具备身份认证的环境运行。

## 产品问题

个人投资工具常把行情、策略、持仓和复盘拆散在不同产品里。用户能看到很多数字，却很难还原某次判断使用了什么数据、当时缺了什么信息，以及最后执行是否偏离原计划。

## 产品方案

本项目把投资过程抽象为可持久化对象：观察项、策略与版本、筛选任务、组合、持仓、交易流水、决策、证据快照、Agent 任务和复盘记录。每个结论保留证据值、日期、来源、完整度、缺失原因和失效条件。

## 差异化

1. 数据源失败时有可解释降级，而不是静默空白或伪造数据。
2. 自然语言策略必须经过 DSL 确认、因子可用性检查和筛选漏斗。
3. 多角色 AI 研究不是聊天群，而是可取消、可重试、可审计的持久化任务。
4. AI 不自动交易；用户确认才形成正式决策。
5. 品牌猫状态与时间、市场和持仓风险联动，在专业工具中保留陪伴感。

## 完整工作流

市场或板块 -> 加入观察池 -> 生成策略 -> 筛选结果 -> 创建模拟组合 -> 导入真实持仓 -> 投资委员会分析 -> 用户确认决策 -> 到期复盘 -> 创建策略修订版本。

## 工程实现

- Flask API 与原生前端，保持轻量部署。
- SQLite 保存策略、持仓、流水、证据、任务、决策和复盘。
- Unified DataProvider 负责重试、主备切换、缓存、字段标准化和来源记录。
- AKShare 主源，efinance、Mootdx、BaoStock、腾讯等降级；Tushare 可选。
- pytest 覆盖数据降级、策略 DSL、持仓导入、冲突处理、交易流水、投委会任务与复盘。

## 产品边界

本项目是研究与记录工具，不连接券商、不自动下单、不承诺收益。无法可靠获得的数据保持缺失并说明原因。

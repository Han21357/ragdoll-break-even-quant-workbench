# 老布偶猫量化工作台交互审计

审计日期：2026-07-21

## 审计范围与方法

- 页面：总览、市场、策略、组合、AI研究员、复盘、数据状态。
- 控件：主导航、页签、按钮、空状态动作、输入框、文件上传、长任务、确认动作和错误重试。
- 判定原则：控件必须产生可见反馈；写操作必须确认；失败必须显示原因；异步任务必须有状态；缺失数据不得用 0 或静态样本冒充。

## 全链路状态

1. 市场或板块加入观察池：已完成，保存当时来源与证据。
2. 观察池生成策略草案：已完成，需用户确认 DSL。
3. 策略筛选：已完成，输出漏斗、逐股通过原因和失败来源。
4. 筛选结果创建模拟组合：已完成，保存策略和筛选证据快照。
5. 模拟组合转真实持仓：已提供导入预览；数量或成本缺失时阻止提交。
6. 真实持仓投委会分析：已完成，角色独立取证并持久化状态。
7. 用户确认决策：已完成，不会自动交易或修改组合。
8. 到期复盘：已完成结构化记录；真实行情不足时保留明确缺失原因。
9. 复盘创建策略版本：已完成，需要用户再次确认。

## 数据与安全边界

- Tushare 仅在配置 Token 后参与增强，不影响项目启动。
- 主源失败后按 DataProvider 注册顺序切换备源；全失败时可返回带 `stale` 标记的本地快照。
- 所有数据结果携带来源、数据日期、更新时间、完整度和错误原因。
- 账户总资产、现金、基准对齐收益或风格因子不可得时，仓位、Beta 和风格暴露显示缺失原因。
- 投资委员会不平均角色分数，主席保留反方意见、证据缺口与可证伪条件。
- AI 输出仅为研究意见，不构成收益承诺，不自动发出交易指令。

## 仍待 P2

- 持仓编辑目前支持确认式 API，页面主要提供导入、清仓和删除；完整抽屉式逐字段编辑器可继续增强。
- 投委会后端支持取消和单角色重试，页面暂以执行状态和失败原因展示为主。
- 真实权益曲线当前以静态持仓复权回溯；历史成交足够后应按逐日现金流重建动态净值。
- Beta 和风格暴露需接入日期一致的沪深300与风格因子序列。
- 机构调研、研报关注度和公司行动仍受免鉴权数据源稳定性约束，失败时明确显示来源错误。

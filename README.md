# 老布偶猫回本之路 · 投资决策记录与复盘工作站

> 用真实行情、结构化观点挑战和事后回查，帮助个人投资者减少凭感觉决策。AI 提供证据，不替用户下单。

## 项目定位

本项目服务于收盘后复盘和投资决策留痕。核心流程不是“让 AI 猜涨跌”，而是：

```text
查看持仓与行情 -> 写下判断 -> AI 提供多空证据和风险质疑
-> 用户确认是否采纳 -> 固化决策与基准价 -> 7 天后回查 -> 总结偏差
```

产品不会连接券商或自动下单。只有用户明确确认的观点才进入决策复盘。

## 核心能力

- **持仓与市场复盘**：集中查看持仓、指数、信号和风险状态
- **AI 观点挑战**：通过威科夫、技术面、基本面、多空和风控角色补充反证
- **人工确认闭环**：AI 输出默认是待核验观点，不自动记作用户决策
- **决策复盘**：固化股票、方向、仓位、置信度、价格和模型/提示词版本
- **研究性效果指标**：买卖观点计算方向调整收益，持有观点单列稳定率
- **组合历史回放**：按确定性等权规则回放指定股票池，不冒充因子策略回测

## 度量口径

方向指标只统计买入和卖出观点：买入使用标的实际收益，卖出使用实际收益的相反数。持有/观望不再混入方向命中率，而是单独计算 7 日波动是否处于 +/-5% 区间。

方向样本少于 30 条时，页面会明确提示样本不足。命中率、方向调整收益和研究夏普仅用于复盘产品行为，不用于证明策略可盈利，也不构成投资建议。

## 技术栈

| 层 | 技术 |
|---|---|
| 前端 | 原生 HTML/CSS/JavaScript，单文件运行 |
| 后端 | Python Flask |
| 行情 | baostock A 股日频数据 |
| AI 编排 | 单次结构化证据审查；实验性 TradingAgents 适配保留在源码中 |
| LLM | OpenAI 兼容接口，模型由环境变量配置 |
| 存储 | 本地 JSON，适合单用户本地使用 |

## 快速开始

```bash
pip install -r requirements.txt
bash start-server.sh
open http://localhost:8766
```

系统默认没有样例持仓。新增持仓时可输入股票代码或简称进行模糊搜索，代码、正式简称和所属市场会由 Baostock 校验并自动补齐；用户只需填写成本和数量。现价来自 Baostock 最新可用日线，行情不可用时会显示缺失状态，不回退到模拟数据。

## 功能面板

| 面板 | 作用 |
|---|---|
| 持仓仪表盘 | 查看真实持仓汇总和市场状态 |
| 持仓明细 | 新增、编辑、删除持仓 |
| 策略矩阵 | 按观察/持有/减仓等状态组织持仓 |
| 信号池 | 汇总量价信号，作为研究输入 |
| 量化选股 | 组合因子并预览候选标的 |
| AI 观点挑战 | 展示多空证据、风险质疑和待确认观点 |
| 组合回放 | 基于真实历史数据检查指定股票池的等权表现 |
| 决策复盘 | 回查已确认观点及其研究性指标 |

## 项目结构

```text
wyckoff-portfolio.html   前端主文件
wyckoff-server.py        Flask API 与异步任务
agent_chain.py           历史实验性多角色链路（主流程不再调用）
ta_adapter.py            TradingAgents 到 baostock 的数据适配
effect_tracker.py        决策记录与回查指标
PRD.md                   产品需求文档
ARCHITECTURE.md          技术架构说明
start-server.sh          本地启动脚本
```

## 产品边界

- 不自动交易，不承诺收益，不把 AI 输出当成事实
- 行情失败时展示不可用状态；模拟数据必须显式标注
- 多角色输出是观点生成与反证机制，不代表多个独立模型达成共识
- 自定义因子未接入真实计算或执行器时会明确失败，不生成随机指标
- 本地 JSON 适合作品集和个人使用，不适合多用户生产环境

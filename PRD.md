# PRD：老布偶猫回本之路

版本 v3.0 · 2026-07-18

## 目标用户

有A股经验、愿意系统化研究策略但不想直接写代码的个人投资者。他们需要把模糊投资想法转成可计算规则，验证历史表现，并在真实或模拟持仓中跟踪策略是否仍然成立。

## 核心工作流

```text
市场雷达发现机会
-> 用户用自然语言表达投资想法
-> 策略编译Agent生成语义拆解和DSL
-> 数据审计Agent展示歧义与缺口
-> 用户确认规则
-> 系统运行筛选漏斗
-> 保存策略版本
-> 回测中心运行真实历史回测
-> 策略体检生成可信度与风险
-> 加入观察池或模拟组合
-> 持仓关联来源策略
-> 复盘模型信号与实际操作偏差
```

## 产品差异化

- 自然语言不是直接出股票名单，而是先出“策略翻译确认页”。
- 策略DSL可校验、可保存、可版本化。
- 数据源状态和样本范围显式展示。
- 回测区分交易级胜率、收益周期正收益率和信号级统计。
- 复盘不替代量化主线，而是策略生命周期闭环的一部分。

## 策略语义编译器

输入示例：

```text
找价格低于100元、最近没有进入下行通道、近5日没有大涨、行业景气较强、机构关注增加的股票。
```

输出三层：

- 用户原始表达：完整保留。
- 语义拆解：如 `close < 100`、`return_5d <= 12%`。
- 可执行DSL：`StrategyDSL` + `ConditionGroup` + `Condition`。

LLM或规则编译器不得生成任意Python。所有操作符和因子必须来自白名单和因子注册表。无法确定的语义必须显示可选定义和当前数据支持情况。

## 策略体检

每次回测完成后生成确定性评分：

- 数据可信度
- 回测可信度
- 参数稳定性
- 成本鲁棒性
- 市场适应性
- 收益分散度

LLM只解释检测结果，不直接改写指标或随意打分。

## 持仓策略血缘

每个持仓可关联来源策略、版本、入选日期、入选排名、入选时因子值、当前条件成立情况和用户是否遵循信号。用户必须能区分模型仍持有、建议减仓、已退出和用户选择继续持有。

## 策略健康中心

策略状态包括：正常、观察、衰减、数据异常、暂停。状态不能只由短期亏损决定，必须综合样本量、偏离程度、持续时间、市场环境和数据健康度。

## 成功指标

- 策略翻译确认完成率。
- 数据可用性检查覆盖率。
- 筛选漏斗可解释率。
- 回测完成率。
- 策略体检风险项采纳率。
- 持仓血缘关联率。
- 到期复盘完成率。

护栏指标：

- 自动下单次数为0。
- 未标注模拟数据出现次数为0。
- 数据失败却生成结果次数为0。
- API Key进入前端或日志次数为0。

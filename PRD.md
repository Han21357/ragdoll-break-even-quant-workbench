# 产品需求文档（PRD）
## 老布偶猫回本之路：投资决策记录与复盘工作站

> 版本 v2.0 · 2026-07-17 · 作者 solo

## 0. 执行摘要

个人投资者通常不缺行情和观点，缺的是一套能长期执行的决策纪律：操作前写清依据和失效条件，操作时主动寻找反证，操作后回看当时信息与结果之间的偏差。

本产品把 AI 放在“观点挑战者”而不是“投资经理”的位置。用户先选择标的并形成初步判断，AI 再从量价、技术面、基本面、多头、空头和风控等角度补充证据与反证。AI 输出默认只是待核验观点，只有用户明确确认后，系统才固化决策、基准价和模型版本，并在观察期结束后生成复盘指标。

核心价值不是提高短期预测胜率，而是提高高质量决策记录率，让用户能回答三个问题：当时为什么这样判断、有哪些反证被忽略、结果出现后应该修正哪条规则。

## 1. 问题定义

### 1.1 目标用户

有 2 年以上 A 股经验、会看基本行情但没有稳定复盘流程的个人投资者。他们愿意听 AI 的观点，但希望看到依据并保留最终决定权。

### 1.2 核心问题

1. 信息分散，用户需要在多个平台之间重复检索和整理。
2. 买卖依据未结构化，事后很难还原当时判断。
3. 用户容易只寻找支持已有结论的信息，缺少强制反证。
4. AI 输出缺少来源、版本和事后回查，容易形成虚假信任。
5. 预测准确率、持有稳定率和策略收益被混算，指标不能支持可靠判断。

### 1.3 Jobs to Be Done

| 情境 | 用户任务 | 预期结果 |
|---|---|---|
| 收盘后查看持仓 | 快速识别需要处理的风险和异常 | 10 分钟内形成待研究清单 |
| 操作前 | 写清投资假设、风险和失效条件 | 决策依据可追溯 |
| 已有倾向时 | 主动寻找反证 | 降低确认偏误和过度自信 |
| AI 给出观点后 | 决定采纳、调整或拒绝 | 最终决定权属于用户 |
| 观察期结束后 | 区分信息错误、规则错误和执行错误 | 形成可复用的复盘结论 |

## 2. 产品定位与边界

### 2.1 一句话定位

一个帮助个人投资者记录依据、挑战观点并回查结果的收盘后复盘工作站。

### 2.2 不做什么

- 不连接券商，不自动下单，不提供收益承诺。
- 不把多 Agent 输出描述为多个独立模型的共识。
- 不将少量样本的方向命中率包装成策略有效性证明。
- 不在真实模式下使用未标注的示例行情、持仓或分析结果。
- 不展示模型的原始思维过程，只展示结构化依据、反证、风险和结论摘要。

## 3. 核心用户流程

```text
查看真实持仓与行情
  -> 选择需要研究的标的
  -> 写下当前判断和风险条件
  -> AI 生成支持证据、反证和风险质疑
  -> 用户确认、修改或拒绝
  -> 确认后固化决策和价格基准
  -> 7 天后回查行情
  -> 区分方向观点与持有观点并生成复盘
  -> 用户补充“判断、规则、执行”层面的反馈
```

AI 运行失败时保留用户已输入内容，并允许重试。未获取到真实基准价时禁止写入可度量记录。

## 4. 功能优先级

| 优先级 | 功能 | 用户价值 | 当前状态 |
|---|---|---|---|
| P0 | 持仓管理与真实数据状态 | 建立可信输入 | 已实现 |
| P0 | AI 观点挑战 | 补充证据与反证 | 已实现 |
| P0 | 人工确认后再记录 | 明确人机责任边界 | 已实现 |
| P0 | 决策记录与到期回查 | 建立复盘闭环 | 已实现 |
| P0 | 数据/演示模式标识 | 防止样例被误认为真实 | 已实现 |
| P1 | 投资假设和失效条件表单 | 提高记录质量 | 待实现 |
| P1 | 采纳/修改/拒绝与原因 | 衡量 AI 的实际辅助价值 | 待实现 |
| P1 | 复盘标签：信息/规则/执行错误 | 形成可复用学习 | 待实现 |
| P1 | 快速模式与深度模式 | 控制等待时间和成本 | 待实现 |
| P2 | 月度决策复盘报告 | 汇总长期行为变化 | 待实现 |

## 5. AI 观点挑战

### 5.1 角色设计

| 环节 | 任务 | 输出 |
|---|---|---|
| 市场扫描 | 提供指数与环境数据 | 数据摘要及时间戳 |
| 量价/技术/基本面 | 从不同分析框架提取信号 | 依据、缺失数据、不确定性 |
| 多头/空头 | 分别寻找支持和反对证据 | 可核验论点列表 |
| 风控 | 检查仓位、止损和集中度风险 | 风险项与限制条件 |
| 综合 | 压缩为待确认观点 | 方向、仓位、置信度、摘要 |

多角色链路的价值是强制覆盖不同视角，不是让系统替用户拍板。最终卡片必须标记“AI 综合观点”“待用户确认”“不会自动交易”。

### 5.2 输出要求

```json
{
  "action": "买入|卖出|持有|观望",
  "position_pct": 25,
  "stop_loss_pct": -8,
  "target_pct": 15,
  "confidence": 7,
  "summary": "主要依据、关键反证和适用条件"
}
```

事实性字段必须来自数据接口或明确标记无法获取。LLM 不能自行补造财务数据、实时行情或用户持仓。

## 6. 决策记录与度量

### 6.1 记录条件

只有同时满足以下条件才建立记录：

1. 用户点击“记录为我的决策”。
2. 股票代码、方向和真实基准价有效。
3. 系统保存模型、提示词和判定规则版本。

### 6.2 指标口径

| 类型 | 口径 | 是否进入方向命中率 |
|---|---|---|
| 买入/加仓 | 方向调整收益 = 标的实际收益 | 是 |
| 卖出/减仓 | 方向调整收益 = 标的实际收益取反 | 是 |
| 持有/观望 | 7 日绝对涨跌小于 5% 记为稳定 | 否，单列稳定率 |

方向命中率 = 方向正确的买卖观点数 / 已回查买卖观点数。

研究夏普使用 7 日方向调整收益按周频年化，只用于产品内复盘。方向样本少于 30 条时，所有方向聚合指标必须展示“样本不足”，不得输出策略有效结论。

### 6.3 已知限制

- 7 个自然日不是完整的长期持有周期。
- 结果受市场环境、标的选择和用户采纳偏差影响。
- 未执行的观点与真实交易收益不是同一概念。
- 本地 JSON 缺少多用户隔离、并发控制和审计能力。

## 7. 成功指标

### 7.1 北极星指标

**完整决策记录率**：包含投资假设、关键反证、失效条件、用户确认和到期复盘的记录数 / 用户创建的全部决策记录数。

- 当前基线：待测
- 设计目标：连续使用 4 周后达到 80%
- 目标性质：产品设计目标，不是已取得的业务结果

### 7.2 过程指标

| 指标 | 定义 | 设计目标 |
|---|---|---|
| AI 任务成功率 | 成功完成观点挑战 / 总启动次数 | >=95% |
| 快速模式耗时 | 启动到可读结果的 P50 | <15 秒 |
| 深度模式耗时 | 启动到完整结果的 P50 | <120 秒 |
| 依据可追溯率 | 有数据来源或缺失标记的事实项占比 | >=90% |
| 决策确认率 | 被确认或修改后记录的 AI 观点占比 | 观察指标，不设越高越好 |
| 到期复盘完成率 | 有用户复盘反馈的到期记录占比 | >=70% |

### 7.3 研究指标

方向命中率、方向调整收益、持有稳定率和研究夏普属于研究指标，不作为北极星指标，也不作为收益承诺。报告时必须同时给出样本量、观察窗和版本。

### 7.4 护栏指标

- 自动下单次数必须为 0。
- 未标注的模拟数据出现次数必须为 0。
- 无真实基准价却进入方向统计的记录数必须为 0。
- 固定测试集上的无依据事实率目标低于 5%。

## 8. 风险与缓解

| 风险 | 缓解措施 |
|---|---|
| LLM 编造事实 | 数据字段与生成字段分离；缺失数据显式展示 |
| 用户过度依赖 AI | 默认待确认；展示反证；禁止自动交易 |
| 行情源失败 | 展示不可用状态并允许重试，不静默回退到未标注样例 |
| 指标误读 | 买卖/持有分开统计；样本下限；显示观察窗和版本 |
| 长链路等待 | 提供快速/深度模式；异步状态；保留输入 |
| 隐私与安全 | 默认本地存储；不上传券商凭证；生产化前增加鉴权和加密 |

## 9. 路线图

### 阶段 A：可信技术原型（已完成）

- 持仓、行情和信号面板
- 组合历史回放面板，使用确定性的等权规则
- 多角色观点挑战链路
- 人工确认后写入决策记录
- 方向/持有分离的复盘口径
- 空状态与显式演示模式

### 阶段 B：完整决策纪律（下一步）

- 投资假设、反证和失效条件表单
- 采纳、修改、拒绝及原因记录
- 到期后的信息/规则/执行偏差标签
- 快速模式与深度模式的真实耗时埋点

### 阶段 C：长期复盘（规划）

- 月度决策质量报告
- 按市场环境、标的和模型版本分层比较
- 数据导出、备份和迁移

上线到多用户环境前，必须补齐鉴权、权限隔离、结构化数据库、行情许可审查和完整风险披露。
